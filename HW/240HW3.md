## HW3: หาตัวอย่าง C# Class 3 ประเภท
a) Utility Class
**System.Math**
1. เป็น Static Class : ถูกประกาศด้วยคีย์เวิร์ด static ซึ่งทำให้ไม่สามารถสร้าง Instance (Object) ของคลาสนี้ได้ (ไม่สามารถ new Math() ได้)
2. รวบรวม Helper Methods : ภายในคลาสประกอบด้วย Static Methods อย่างเดียว ที่ใช้สำหรับการคำนวณทางคณิตศาสตร์ทั่วไป ฟังก์ชันนี้ทำงานจบในตัวและไม่ขึ้นกับสถานะ (State) ของ object ใด ๆ
3. ไม่มีการเก็บ State : ไม่มีการเก็บข้อมูลสถานะของ Instance เอาไว้ เป็นเพียงที่รวมฟังก์ชันอำนวยความสะดวกเท่านั้น

b) Mutable Class
**System.Text.StringBuilder**
1. เปลี่ยนแปลงสถานะภายในได้ (Modifiable State) : เมื่อเรียกใช้เมธอดเช่น .Append(), .Remove(), หรือ .Insert() คลาสนี้จะทำการแก้ไขข้อมูลในหน่วยความจำเดิม (Buffer) ของ Object นั้น ๆ โดยตรง
2. ไม่สร้าง Object ใหม่ : การแก้ไขข้อความจะไม่ส่งคืน Instance ใหม่ทุกครั้ง (ต่างจาก String) แต่จะคืนค่าเป็น Reference เดิมที่ได้รับการแก้ไขแล้ว ทำให้ค่า Properties เช่น Length หรือ Capacity เปลี่ยนแปลงไปตามการใช้งานจริง
3. ออกแบบมาเพื่อการแก้ไข : ถูกออกแบบมาแก้ปัญหาประสิทธิภาพเมื่อต้องมีการแก้ไขข้อความบ่อยๆ โดยยอมให้สถานะของวัตถุไม่คงที่

c) Immutable Class
**System.String**
1. แก้ไขไม่ได้หลังสร้าง (Read-only State) : หลังจากที่ String ถูกสร้างขึ้น ข้อมูลตัวอักษรภายใน (Char array) จะถูกกำหนดตายตัวและไม่สามารถเปลี่ยนแปลงได้อีก
2. การแก้ไขคือการสร้างใหม่ : เมื่อเราเรียกใช้เมธอดที่ดูเหมือนจะแก้ไขข้อความ เช่น .ToUpper(), .Substring(), หรือ .Replace() ระบบจะไม่แก้ที่ตัวเดิม แต่จะจัดสรรหน่วยความจำใหม่และส่งคืน Object ใหม่ (New Instance) ที่มีค่าตามที่ต้องการออกมาแทน ส่วน Object เดิมจะยังคงมีค่าเหมือนเดิม
3. ความปลอดภัย (Thread Safety) : เนื่องจากสถานะไม่สามารถเปลี่ยนได้ จึงปลอดภัยเมื่อถูกใช้งานพร้อมกันหลาย Thread โดยไม่ต้องกังวลว่าค่าจะถูกเปลี่ยนกลางคัน
